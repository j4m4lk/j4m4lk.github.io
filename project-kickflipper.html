Project KickFlipper

Mobile soccer trickshot game – devlog, screenshots, and optimizations

Status: In Development · Engine: Unity · Platforms: Android / iOS

Unity Netcode (Authoritative) Mobile Optimization Procedural Levels

KickFlipper is a fast, score‑chasing trickshot game. Curve the ball, bank off walls for multipliers, and chain targets for big combos. Below I’m tracking how I’m reducing CPU/GPU cost, memory, and draw calls while keeping the feel crispy on mid‑range phones.

Overview

Core loop: aim → kick → hit targets → chase multipliers (bank shots, curve, perfect timing). Levels are bite‑sized (30–90s) and tuned for one‑hand play.

  • Authoritative setup with Unity Netcode QuickSetup.
  • Deterministic scoring with server validation for combo/multiplier events.
  • Procedural target layouts with difficulty bands and seed sharing.
  • Low‑overhead UI with pooled widgets and batched TMP updates.

Optimization Log

Batching & Draw Calls

Goal: Reduce overdraw + state changes on mid‑tier Android GPUs.

  • Enabled GPU Instancing for repeat props (+ SRP Batcher).
  • Merged static level pieces (Static Batching) where memory allowed.
  • Trimmed materials from 24 → 9. Unified tiling and roughness maps.

Result: Heavy scene draw calls: ~1200 → ~380.

Physics & FixedUpdate Cost
  • Moved non‑critical checks to Update using time budget + counters.
  • Pooled colliders for temporary targets to avoid allocations.
  • Reduced solver iterations on mobile profile where safe.

Result: FixedUpdate CPU: ‑35–45% on Pixel 6 / Galaxy A53.

UI & TMP
  • Pooled TMP objects; disabled auto‑size; batched font atlases.
  • Minimized layout rebuilds; deferred stat updates to 5–10 Hz.

Result: Layout + Canvas rebuild spikes eliminated in end‑of‑round screen.
